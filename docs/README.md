# Документация Heroes Online

Центральное оглавление всей проектной документации.

Документы организованы по разделам с числовыми префиксами — это задаёт порядок чтения и приоритет. Начинайте с `00_Project`, затем переходите к тематическим разделам по мере необходимости.

---

## Как пользоваться

| Правило | Описание |
|---------|----------|
| **Порядок** | Файлы внутри раздела нумеруются (`00_`, `01_`, …) — читайте сверху вниз |
| **Актуальность** | При расхождении между разделами приоритет у более специфичного документа |
| **Решения** | Архитектурные и проектные решения фиксируются в `14_Decisions/` |
| **Изменения** | Ход разработки — в `13_DevLog/` |

## Сайт документации

Документация публикуется как сайт через [MkDocs](https://www.mkdocs.org/). Конфигурация — `mkdocs.yml` в корне репозитория.

```bash
pip install -r requirements-docs.txt
mkdocs serve
```

Автоматическая публикация через GitHub Actions будет добавлена позже.
