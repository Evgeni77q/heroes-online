# Руководство для участников

Спасибо за интерес к Heroes Online! Этот документ описывает, как участвовать в разработке.

## Перед началом

1. Ознакомьтесь с [документацией](docs/README.md)
2. Прочитайте раздел [00 — Проект](docs/00_Project/index.md): видение, конституция, стандарты
3. Настройте окружение (см. ниже)

## Настройка окружения

```bash
# Клонировать репозиторий
git clone <repo-url>
cd heroes-online

# Скопировать переменные окружения
cp .env.example .env

# Запустить инфраструктуру для разработки
docker compose up -d
```

## Документация

- Исходники — в каталоге `docs/`
- Локальный просмотр сайта:

```bash
pip install -r requirements-docs.txt
mkdocs serve
```

- При изменении документации обновляйте `docs/README.md`, если меняется структура
- Архитектурные решения фиксируйте в `docs/14_Decisions/`

## Спецификации

Формальные спецификации (API, протоколы, контракты) размещаются в каталоге [`specs/`](specs/).

## Стиль коммитов

Используйте понятные сообщения на русском или английском:

```
тип: краткое описание

feat: добавить авторизацию персонажа
fix: исправить синхронизацию позиции
docs: описать эндпоинт /api/characters
```

Типы: `feat`, `fix`, `docs`, `refactor`, `test`, `chore`.

## Pull Request

1. Создайте ветку от `main`: `feat/описание` или `fix/описание`
2. Внесите изменения, следуя [стандартам проекта](docs/00_Project/04_Project_Standards.md)
3. Убедитесь, что тесты проходят (когда появятся)
4. Обновите документацию, если поведение изменилось
5. Опишите изменения в PR и свяжите с issue, если есть

## Сообщения об уязвимостях

Не создавайте публичные issue для проблем безопасности. См. [SECURITY.md](SECURITY.md).

## Вопросы

Если что-то неясно — создайте issue с меткой `question` или обратитесь к мейнтейнерам.
