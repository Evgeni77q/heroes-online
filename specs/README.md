# Спецификации

Формальные спецификации проекта: API, протоколы, контракты данных, схемы.

## Назначение

| Тип | Пример | Описание |
|-----|--------|----------|
| API | `api/openapi.yaml` | REST/gRPC контракты |
| Protocol | `protocol/game-sync.md` | Игровые протоколы синхронизации |
| Data | `data/entities.json` | Схемы сущностей и сообщений |
| Events | `events/domain-events.md` | Доменные события |

## Связь с документацией

- Описательная документация — в [`docs/`](../docs/)
- Формальные контракты — здесь, в `specs/`

## Правила

1. Спецификация должна быть машиночитаемой, где это возможно (OpenAPI, JSON Schema, Protobuf)
2. Каждое изменение контракта — отдельный коммит или PR с обновлением версии
3. Breaking changes фиксируются в [CHANGELOG.md](../CHANGELOG.md)

## Структура (план)

```
specs/
├── README.md           ← вы здесь
├── api/                OpenAPI / gRPC
├── protocol/           Игровые протоколы
├── data/               JSON Schema, модели
└── events/             Доменные события
```

*Подкаталоги будут созданы по мере появления спецификаций.*
