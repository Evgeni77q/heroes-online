# Политика безопасности

## Поддерживаемые версии

| Версия | Поддержка        |
|--------|------------------|
| latest | Активная разработка |

> Проект на ранней стадии. Политика будет уточняться по мере выхода релизов.

## Сообщить об уязвимости

**Не публикуйте уязвимости в открытых issue или обсуждениях.**

Сообщите напрямую одним из способов:

- Email: `security@heroes-online.dev` *(замените на актуальный адрес)*
- GitHub Security Advisory: *будет доступно после публикации репозитория*

### Что включить в отчёт

- Описание уязвимости
- Шаги для воспроизведения
- Возможное влияние
- Предложения по исправлению (если есть)

### Сроки ответа

| Этап | Срок |
|------|------|
| Подтверждение получения | 3 рабочих дня |
| Первичная оценка | 7 рабочих дней |
| Исправление (критические) | по согласованию |

## Область

В рамках политики:

- Backend и API
- Frontend-клиент
- Инфраструктура и конфигурация деплоя
- Аутентификация и авторизация
- Игровые протоколы и синхронизация

Вне области:

- Социальная инженерия
- DoS без эксплуатируемой уязвимости
- Проблемы в сторонних зависимостях без подтверждённого влияния на проект

## Рекомендации

- Не храните секреты в репозитории
- Используйте `.env` локально, никогда не коммитьте его
- Следуйте [CONTRIBUTING.md](CONTRIBUTING.md) при разработке
